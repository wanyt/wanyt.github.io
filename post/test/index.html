ThreadLocalMap & ThreadLocal

#默认分类/数据结构 #博客

线性探测法解决哈希冲突的典型实现

ThreadLocalMap是ThreadLocal的内部类，用来实现ThreadLocal存取线程局部变量的功能。ThreadLocalMap用来存储key-value形式的键值对，是Map的简易实现形式，但是它没有实现Map接口，不属于Java的集合体系，它是包级别可见的，没有对开发者开放，仅供ThreadLocal和Thread使用。

todo 哈希表是什么？

使用哈希表就必须要处理哈希冲突，常见的处理哈希冲突的方式包含链地址法和线性探测法。Java中常用的HashMap使用链地址法（也叫拉链法）解决哈希冲突；而Hashtable、WeakHashMap和本篇分析的ThreadLocalMap则使用了线性探测法解决哈希冲突。

什么是线性探测法

static class ThreadLocalMap {
	private static final int INITIAL_CAPACITY = 16;   // 1
	private Entry[] table;     // 2
	private int size = 0;      // 3
	private int threshold;     // 4

	private void setThreshold(int len) {  
		threshold = len * 2 / 3;
	}

	private static int nextIndex(int i, int len) {  // 5
		return ((i + 1 < len) ? i + 1 : 0);
	}
	private static int prevIndex(int i, int len) {  // 6
		return ((i - 1 >= 0) ? i - 1 : len - 1);
	}
}
  1. ThreadLocalMap的初始容量是16
  2. 键值对以Entry的形式存放在Entry数组中
  3. size是表中元素的数量
  4. 触发扩容的阈值，每当size的值达到threshold时，会触发扩容，每次扩容会计算触发下次扩容的值，该值是表的总容量的三分之二
  5. 获取 i 处的下一个索引
  6. 获取 i 处的上一个索引

扩容会创建双倍容量的数组，然后把原始数组的元素依次rehash到新数组中，不仅浪费空间而且浪费时间，非常消耗性能。平衡这种消耗的有效办法是，根据使用场景设置表的初始容量，从而避免频繁的扩容。

在ThreadLocalMap中Entry是键值对的具体表现，存储在Entry数组中。它继承WeakReference，使用ThreadLocal作为值：

static class Entry extends WeakReference<ThreadLocal<?>> {
	Object value;
	Entry(ThreadLocal<?> k, Object v) {
		super(k);
		value = v;
	}
}

Entry之所以使用WeakReference，是因为Entry的设计要有利于GC回收，使不再被使用的变量及时从内存中释放。某一线程中创建的所有ThreadLocal都存放在Thread.threadLocals中，Thread.threadLocals是ThreadLocalMap类型的变量专门用来

存储该线程所有的ThreadLocal，当ThreadLocal在某线程中非常广泛的创建并使用的同时，它也占用了非常多的内存，这时需要有一种机制去判断哪些ThreadLocal可以回收，这时WeakReference来了。

Java文档对WeakReference的用途做了十分清晰的描述：弱引用通常用来实现规范的数据映射（mapping）。

Java中使用new创建的强引用对象在回收时通常要经历多个流程，比如，使用标记-清除或者标记压缩GC算法，在GC运行时对象会先被标记，GC再次运行时才会清除被标记的对象。而对于WeakReference，GC在运行时一旦判定该对象为弱可达或不可达，则直接将该对象回收。

如何解释弱可达和不可达？ 弱可达：WeakReference被另外一个WeakReference引用 不可达：没有任何类型的引用 引用WeakReference对象

下面开始分析ThreadLocalMap的添加数据，对于一个哈希表来说，插入数据分为两个步骤：1. 通过哈希函数获取key在哈希表中的索引，2. 把数据插入到该索引中，并处理哈希冲突。

ThreadLocalMap也不例外，首先通过哈希函数获取key在哈希表中对应的索引值，然后获取哈希表中该索引下的元素，如果元素为空，则根据key-value构建Entry，把Entry插入到该索引下；如果元素不为空，则从该索引向下搜索，直到发现为null的

private void set(ThreadLocal<?> key, Object value) {
	Entry[] tab = table;
	int len = tab.length;
	int i = key.threadLocalHashCode & (len-1); // 获取key的哈希值，并计算该元素在数组中的索引

	for (Entry e = tab[i];
		 e != null; // 如果e有值，则进入循环；如果e为null，则退出循环
		 e = tab[i = nextIndex(i, len)]) { // 循环体执行完成后，执行本语句；执行本语句的前提是：i-1有值
		ThreadLocal<?> k = e.get();

		if (k == key) { // 找到了相同的key，则新值覆盖旧值，然后返回
			e.value = value;
			return;
		}

		if (k == null) { // Entry不为空，但key为null，代表key已被GC回收
			replaceStaleEntry(key, value, i);
			return;
		}
	}

	// 执行到此代表哈希后的数组索引处没有Entry，可直接将key-value存储至该索引处
	tab[i] = new Entry(key, value);
	int sz = ++size; // 容量+1
	if (!cleanSomeSlots(i, sz) && sz >= threshold) 
		// 如果数组中不需要清除已被回收的元素，并且容量已达扩容的阈值，则对数组进行扩容
		rehash();
}

private void replaceStaleEntry(ThreadLocal<?> key, Object value,int staleSlot) {
	Entry[] tab = table;
	int len = tab.length;
	Entry e;

	int slotToExpunge = staleSlot;
	//从staleSlot开始向上遍历
	for (int i = prevIndex(staleSlot, len); //获取当前位置的前一个位置作为初始索引
		 (e = tab[i]) != null; // 如果entry有值，进入循环体，如果为null则退出循环
		 i = prevIndex(i, len)) // 获取前一个索引的值
		if (e.get() == null) // 循环体，如果Entry不为null且key为null，则代表key已被GC回收，该索引处的Entry可被销毁
			slotToExpunge = i; //记录可以被销毁的索引

	//从staleSlot开始向下遍历
	for (int i = nextIndex(staleSlot, len); //获取当前位置的下一个索引作为初始值
		(e = tab[i]) != null; //如果entry有值，进入循环体，如果为null则退出循环
		i = nextIndex(i, len)) { //获取下一个索引
		ThreadLocal<?> k = e.get();

		if (k == key) { //如果索引处的Entry.key和方法参数中的key相等
			e.value = value;//新值覆盖旧值

			tab[i] = tab[staleSlot];  //!!!!!!!此处代码意图还不是太明晰!!!!!!!
			tab[staleSlot] = e;

			if (slotToExpunge == staleSlot)
				slotToExpunge = i;
			cleanSomeSlots(expungeStaleEntry(slotToExpunge), len);
			return;
		}

		if (k == null && slotToExpunge == staleSlot)
			slotToExpunge = i;
	}

	tab[staleSlot].value = null; //标记旧的Entry可被GC回收
	tab[staleSlot] = new Entry(key, value); //记录新的Entry

	if (slotToExpunge != staleSlot) 
	// 如果slotToExpunge和staleSlot的值不相等，则代表哈希表中有需要被销毁的Entry，此时需要做清理Entry的操作
		cleanSomeSlots(expungeStaleEntry(slotToExpunge), len);
}

private boolean cleanSomeSlots(int i, int n) {
	boolean removed = false; // 是否有被移除的Entry
	Entry[] tab = table;
	int len = tab.length;
	// 先执行do，然后执行while，满足条件再执行do；如此往复
	do {
		i = nextIndex(i, len); //获取下一个索引
		Entry e = tab[i]; //获取下一个索引处的Entry
		if (e != null && e.get() == null) { // 如果Entry不为null,但key为null,则该Entry可被销毁
			n = len; // 控制遍历次数
			removed = true; 
			i = expungeStaleEntry(i); //删除该处索引的Entry

		}
	} while ( (n >>>= 1) != 0); //每次n右移1位则等同于n除以2，！！为什么这样做有待考证！！
	return removed; 
}

private int expungeStaleEntry(int staleSlot) {
	Entry[] tab = table;
	int len = tab.length;

	tab[staleSlot].value = null;
	tab[staleSlot] = null; //把索引处的Entry置为null
	size--; // size减1

	// 从staleSlot+1开始向下遍历, 
	// 如果发现有Entry.key为null, 则将该Entry置为null
	// 如果Entry不为null, 则使用哈希值对其重新映射
	Entry e;
	int i;
	for (i = nextIndex(staleSlot, len); // 从staleSlot+1开始
		 (e = tab[i]) != null;  // Entry有值进入循环, 否则退出循环
		 i = nextIndex(i, len)) { // 获取下一个索引, 向下遍历
		ThreadLocal<?> k = e.get();
		if (k == null) { // 如果key为null将Entry置为null
			e.value = null;
			tab[i] = null;
			size--;
		} else { // 如果key不为null, 对Entry rehash
			int h = k.threadLocalHashCode & (len - 1);
			if (h != i) {
				tab[i] = null;

				while (tab[h] != null)
					h = nextIndex(h, len); // 线性探测解决哈希冲突
				tab[h] = e;
			}
		}
	}
	return i;
}

ThreadLocalMap是ThreadLocal的静态内部类，它实现了哈希表，以键值对的形式存储线程局部变量。为了便于处理大量的、长生命周期的对象，他使用WeakReference作为键，但是，因为它没有使用ReferenceQueue导致只有当内存 空间不足时才会回收不再使用的Entry。
